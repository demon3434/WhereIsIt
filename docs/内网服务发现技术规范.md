# WhereIsIt 内网服务发现技术规范（当前生效版）

## 1. 当前唯一推荐部署方案

本项目在 Linux 服务器上采用：
- Docker Compose 使用 `bridge` 网络运行应用
- Avahi Reflector 在宿主机反射 mDNS 组播

对应文件：
- `docker-compose.avahi.yml`
- `docker-compose.avahi.self-build.yml`
- `scripts/setup_avahi_whereisit.sh`

## 2. 关键设计

### 2.1 网络设计

- Compose 网络名固定：`whereisit_mdns`
- 宿主机 Linux bridge 网卡固定：`br-whereisit`
- Avahi 允许接口：`<默认出口网卡>,br-whereisit`

说明：
- `whereisit_mdns` 是 Docker 网络名
- `br-whereisit` 是宿主机网卡名（Avahi 使用这个）

### 2.2 服务发现协议参数

- 协议：mDNS / DNS-SD
- 服务类型：`_whereisit._tcp.local.`
- Android 扫描类型：`_whereisit._tcp`
- 服务实例名：`WhereIsIt`
- 广播地址：`SERVICE_ADVERTISE_HOST`（必须填写宿主机 LAN IP）
- 广播端口：默认跟随 `WEB_PORT`（可选 `SERVICE_ADVERTISE_PORT` 覆盖）

## 3. 一键部署步骤（用户视角）

1. 配置 `.env`（至少确认以下参数）
```env
WEB_PORT=3000
SERVICE_DISCOVERY_ENABLED=true
SERVICE_DISCOVERY_TYPE=_whereisit._tcp.local.
SERVICE_DISCOVERY_NAME=WhereIsIt
SERVICE_ADVERTISE_HOST=192.168.1.50
```

2. 配置 Avahi（只需一次）
```bash
sudo bash scripts/setup_avahi_whereisit.sh
```

3. 启动服务（二选一）
```bash
docker compose -f docker-compose.avahi.yml up -d
```

```bash
docker compose -f docker-compose.avahi.self-build.yml up -d --build
```

## 4. 验证标准

### 4.1 服务器侧

执行：
```bash
avahi-browse -atr | grep -i whereisit
```

出现类似输出即视为发现成功：
```text
+ br-whereisit IPv4 WhereIsIt _whereisit._tcp local
= br-whereisit IPv4 WhereIsIt _whereisit._tcp local
  hostname = [whereisit.local]
```

### 4.2 客户端侧

- Android 使用 `NsdManager` 扫描 `_whereisit._tcp`
- 解析到 host/port 后访问：
  - `GET http://<host>:<port>/api/health`
- 返回以下 envelope 视为端到端成功：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "ok"
  }
}
```

## 5. 常见问题

### 5.1 `network ... exists but was not created by compose`

原因：网络由脚本预创建，但 compose 试图按“内部网络”创建。  
处理：`docker-compose.avahi.yml` 已使用 `external: true`，复用已有网络。

### 5.2 填错 `SERVICE_ADVERTISE_HOST`

必须填宿主机局域网 IP。  
不要填：
- `127.0.0.1`
- `172.x.x.x` 这类 Docker 内网地址

### 5.3 Android 扫描不到

按顺序检查：
1. `avahi-daemon` 是否正常
2. `avahi-browse` 是否能看到 `WhereIsIt`
3. Android 是否持有 `MulticastLock`
4. Android 是否用 `_whereisit._tcp` 扫描
