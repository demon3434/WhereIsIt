# WhereIsIt 数据字典

本文基于当前代码中的 `app/models.py` 和 `app/main.py` 启动迁移逻辑整理，重点覆盖业务核心表以及本次新增的图片展示序号能力。

## 1. `users` 用户表

字段：
- `id`：主键
- `username`：登录账号，唯一
- `password_hash`：密码哈希
- `nickname`：昵称
- `full_name`：姓名
- `role`：角色，`admin` / `user`
- `is_active`：是否启用
- `default_house_id`：默认房屋 ID，可空，关联 `houses.id`
- `created_at`：创建时间

## 2. `houses` 房屋表

字段：
- `id`：主键
- `name`：房屋名称，全局唯一
- `sort_order`：展示排序号
- `is_active`：是否启用
- `created_at`：创建时间

## 3. `user_accessible_houses` 用户可访问房屋关系表

字段：
- `user_id`：关联 `users.id`
- `house_id`：关联 `houses.id`

约束：
- 复合主键：`(user_id, house_id)`

## 4. `categories` 分类表

字段：
- `id`：主键
- `user_id`：所属用户
- `name`：分类名称
- `sort_order`：展示排序号
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name)`

## 5. `tags` 标签表

字段：
- `id`：主键
- `user_id`：所属用户
- `name`：标签名称
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name)`

## 6. `locations` 位置/房间表

字段：
- `id`：主键
- `user_id`：所属用户
- `house_id`：所属房屋 ID，可空
- `parent_id`：父级位置 ID，可空，自关联 `locations.id`
- `sort_order`：排序号
- `name`：位置/房间名称
- `path`：完整路径，例如 `A房-客厅-电视柜`
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name, parent_id)`

## 7. `items` 物品表

字段：
- `id`：主键
- `user_id`：所属用户
- `name`：物品名称
- `description`：描述
- `location_detail`：更细粒度的位置描述
- `quantity`：数量
- `brand`：品牌
- `category_id`：分类 ID，可空
- `location_id`：位置/房间 ID，可空
- `created_at`：创建时间
- `updated_at`：最后修改时间
- `voice_terms_dirty_at`：语音搜索词条待刷新时间
- `voice_terms_last_indexed_at`：语音搜索词条最后索引时间

关联：
- 1:N `item_images`
- N:M `tags`，通过 `item_tags`

## 8. `item_tags` 物品-标签关系表

字段：
- `item_id`：关联 `items.id`
- `tag_id`：关联 `tags.id`

约束：
- 复合主键：`(item_id, tag_id)`

## 9. `item_images` 物品图片表

字段：
- `id`：主键
- `item_id`：所属物品 ID，关联 `items.id`
- `filename`：图片文件名
- `url`：图片访问地址
- `display_order`：展示序号，按同一物品内从 `1` 开始连续递增
- `created_at`：图片记录创建时间

说明：
- 前端展示、详情弹窗、编辑页拖拽排序均使用 `display_order`
- 新建数据库时该字段默认存在
- 由于当前系统不支持图片内容编辑，图片“更新时间”业务上等同于 `created_at`
- 老数据库启动时会自动补列，并对历史数据按“同一物品内图片的 `created_at`”回填 `display_order`
- 编辑物品图片顺序、删除图片后，系统会重新整理同一物品下的图片序号，保证连续

## 10. `voice_search_terms` 语音搜索预计算词条表

字段：
- `id`：主键
- `user_id`：所属用户
- `item_id`：所属物品
- `term`：预计算词条
- `term_type`：词条类型
- `weight`：权重
- `created_at`：创建时间
- `updated_at`：更新时间

约束：
- 唯一约束：`(user_id, item_id, term, term_type)`

## 11. `operation_logs` 操作日志表

字段：
- `id`：主键
- `user_id`：所属用户
- `action`：操作类型
- `details`：操作详情
- `created_at`：创建时间

## 12. 核心关系概览

- `users` 与 `houses`：N:M，通过 `user_accessible_houses`
- `houses` 与 `locations`：1:N
- `categories` 与 `items`：1:N
- `locations` 与 `items`：1:N
- `items` 与 `item_images`：1:N
- `items` 与 `tags`：N:M，通过 `item_tags`
- `items` 与 `voice_search_terms`：1:N
