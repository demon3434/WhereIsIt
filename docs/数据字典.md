# WhereIsIt 数据字典

本文基于当前代码 `app/models.py` 与数据库初始化/迁移逻辑 `app/main.py:run_postgres_migrations` 整理。

## 1. `users` 用户表
字段：
- `id`：主键，`int`
- `username`：登录账号，唯一
- `password_hash`：密码哈希
- `nickname`：昵称
- `full_name`：姓名
- `role`：角色，`admin` / `user`
- `is_active`：是否启用
- `default_house_id`：默认房屋 ID，外键到 `houses.id`，可空
- `created_at`：创建时间

关联：
- 1:N `categories.user_id`
- 1:N `tags.user_id`
- 1:N `locations.user_id`
- 1:N `items.user_id`
- 1:N `operation_logs.user_id`
- N:M `houses`，通过 `user_accessible_houses`

## 2. `houses` 房屋表
字段：
- `id`：主键
- `name`：房屋名称，全局唯一，约束 `uq_house_name`
- `sort_order`：排序号
- `is_active`：是否启用
- `created_at`：创建时间

关联：
- 1:N `users.default_house_id`
- N:M `users`，通过 `user_accessible_houses`
- 1:N `locations.house_id`

## 3. `user_accessible_houses` 用户可访问房屋关联表
字段：
- `user_id`：外键到 `users.id`
- `house_id`：外键到 `houses.id`

约束：
- 复合主键：`(user_id, house_id)`

用途：
- 控制普通用户可访问的房屋范围

## 4. `categories` 分类表
字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `name`：分类名
- `sort_order`：排序号
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name)`，约束名 `uq_category_user_name`

关联：
- 1:N `items.category_id`，删除分类时 `items.category_id` 置空

## 5. `tags` 标签表
字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `name`：标签名
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name)`，约束名 `uq_tag_user_name`

关联：
- N:M `items`，通过 `item_tags`

## 6. `locations` 位置/房间表
说明：
- 当前“房间管理”使用本表实现，对外接口由 `app/routers/rooms.py` 提供

字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `house_id`：外键到 `houses.id`，可空
- `parent_id`：自关联外键到 `locations.id`，可空
- `sort_order`：房屋内排序号
- `name`：位置/房间名称
- `path`：路径文本，例如 `瑞金新村-客厅`
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name, parent_id)`，约束名 `uq_location_user_name_parent`

关联：
- 1:N `items.location_id`，删除位置时 `items.location_id` 置空
- 1:N 自关联：`parent_id -> id`

## 7. `items` 物品表
字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `name`：物品名称
- `description`：描述
- `location_detail`：更细粒度的位置描述
- `quantity`：数量
- `brand`：品牌
- `category_id`：外键到 `categories.id`，可空
- `location_id`：外键到 `locations.id`，可空
- `created_at`：创建时间
- `updated_at`：更新时间
- `voice_terms_dirty_at`：语音搜索预计算词条的待刷新时间戳。新增或修改物品时会更新为当前时间
- `voice_terms_last_indexed_at`：该物品最近一次完成语音搜索预计算索引刷新的时间

关联：
- N:1 `users`
- N:1 `categories`
- N:1 `locations`
- 1:N `item_images`
- N:M `tags`，通过 `item_tags`

## 8. `item_tags` 物品-标签关联表
字段：
- `item_id`：外键到 `items.id`
- `tag_id`：外键到 `tags.id`

约束：
- 复合主键：`(item_id, tag_id)`

## 9. `item_images` 物品图片表
字段：
- `id`：主键
- `item_id`：外键到 `items.id`
- `filename`：文件名
- `url`：访问地址
- `created_at`：创建时间

关联：
- N:1 `items`

## 10. `voice_search_terms` 语音搜索预计算词表
说明：
- 这是语音搜索的 PG 预计算表
- 用于把每一个物品可参与语音检索的品牌词、名称片段、分类片段、标签片段预先落库
- 查询阶段不再为每次语音搜索全量扫描物品生成词表，而是直接读取这张表

字段：
- `id`：主键
- `user_id`：所属用户，外键到 `users.id`
- `item_id`：所属物品，外键到 `items.id`
- `term`：预计算出来的词条
- `term_type`：词条类型，当前主要有：
  - `brand`
  - `fragment`
- `weight`：词条权重，用于表达品牌词、名称词、标签词的重要程度
- `created_at`：创建时间
- `updated_at`：更新时间

约束：
- 唯一约束：`(user_id, item_id, term, term_type)`，约束名 `uq_voice_search_term_item_term`

索引：
- `ix_voice_search_terms_user_id`
- `ix_voice_search_terms_item_id`
- `ix_voice_search_terms_term_type`
- `ix_voice_search_terms_term`
- `ix_items_voice_terms_dirty_at`

生成与更新机制：
- 日常运行时，物品新增、修改会把对应 `items.voice_terms_dirty_at` 更新为“当前时间”
- 后台索引 worker 会周期性扫描达到刷新条件的脏数据，只对这些物品做增量刷新
- 刷新条件为：`now - voice_terms_dirty_at >= VOICE_TERMS_INDEX_DELAY_SECONDS`
- 如果在等待窗口内同一物品再次被新增/修改，会把 `voice_terms_dirty_at` 改成更晚的时间，相当于重新计时
- 删除物品时会同步删除该物品在 `voice_search_terms` 中的词条
- 新部署服务端启动时，如果发现已有物品数量与已索引物品数量不一致，会先把全部物品标记为 dirty；随后仍由后台 worker 分批增量补齐，不会在启动阶段直接做一次大规模全量重建

用途：
- 为语音搜索提供品牌词优先匹配
- 为轻量关键词拆解提供名称/分类/标签片段候选
- 为后续搜索排序、错词纠正、别名扩展保留结构基础

## 11. `operation_logs` 操作日志表
字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `action`：操作类型
- `details`：操作详情
- `created_at`：创建时间

关联：
- N:1 `users`

## 12. 关键关系总览
- 用户与房屋权限：`users` N:M `houses`，通过 `user_accessible_houses`
- 用户默认房屋：`users.default_house_id -> houses.id`
- 房屋与房间：`houses` 1:N `locations`
- 分类与物品：`categories` 1:N `items`
- 房间与物品：`locations` 1:N `items`
- 标签与物品：`tags` N:M `items`，通过 `item_tags`
- 物品与语音搜索词条：`items` 1:N `voice_search_terms`
- 用户与业务数据：`users` 1:N `categories/tags/locations/items/operation_logs`
