# WhereIsIt 数据字典

本文基于当前代码 `app/models.py` 与迁移逻辑 `app/main.py:run_postgres_migrations` 整理。

## 1. users（用户表）

字段：
- `id`：主键，`int`
- `username`：登录账号，唯一
- `password_hash`：密码哈希
- `nickname`：昵称
- `full_name`：姓名
- `role`：角色（`admin` / `user`）
- `is_active`：是否启用
- `default_house_id`：默认房屋 ID，外键到 `houses.id`，可空
- `created_at`：创建时间

关联：
- 1:N `categories.user_id`
- 1:N `tags.user_id`
- 1:N `locations.user_id`
- 1:N `items.user_id`
- 1:N `operation_logs.user_id`
- N:M `houses`（通过 `user_accessible_houses`）

## 2. houses（房屋表）

字段：
- `id`：主键
- `name`：房屋名称，全局唯一（`uq_house_name`）
- `sort_order`：排序号
- `is_active`：是否启用
- `created_at`：创建时间

关联：
- 1:N `users.default_house_id`（被引用）
- N:M `users`（通过 `user_accessible_houses`）
- 1:N `locations.house_id`

## 3. user_accessible_houses（用户可访问房屋关联表）

字段：
- `user_id`：外键到 `users.id`
- `house_id`：外键到 `houses.id`

约束：
- 复合主键：`(user_id, house_id)`

用途：
- 控制普通用户可访问的房屋范围。

## 4. categories（分类表）

字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `name`：分类名
- `sort_order`：排序号
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name)`（`uq_category_user_name`）

关联：
- 1:N `items.category_id`（被 `items` 引用，删除分类时置空）

## 5. tags（标签表）

字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `name`：标签名
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name)`（`uq_tag_user_name`）

关联：
- N:M `items`（通过 `item_tags`）

## 6. locations（位置/房间表）

说明：
- 当前“房间管理”使用本表实现，接口由 `app/routers/rooms.py` 提供（前缀 `/api/rooms`）。

字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `house_id`：外键到 `houses.id`，可空（房间场景通常不为空）
- `parent_id`：自关联外键到 `locations.id`，可空
- `sort_order`：房屋内排序号
- `name`：位置/房间名称
- `path`：路径文本（如 `房屋-房间`）
- `is_active`：是否启用
- `created_at`：创建时间

约束：
- 唯一约束：`(user_id, name, parent_id)`（`uq_location_user_name_parent`）

关联：
- 1:N `items.location_id`（被 `items` 引用，删除位置时置空）
- 1:N 自关联（`parent_id` -> `id`）

## 7. items（物品表）

字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `name`：物品名称
- `description`：描述
- `location_detail`：具体位置描述（文本）
- `quantity`：数量
- `brand`：品牌
- `category_id`：外键到 `categories.id`，可空
- `location_id`：外键到 `locations.id`，可空
- `created_at`：创建时间
- `updated_at`：更新时间

关联：
- N:1 `users`
- N:1 `categories`
- N:1 `locations`
- 1:N `item_images`
- N:M `tags`（通过 `item_tags`）

## 8. item_tags（物品-标签关联表）

字段：
- `item_id`：外键到 `items.id`
- `tag_id`：外键到 `tags.id`

约束：
- 复合主键：`(item_id, tag_id)`

## 9. item_images（物品图片表）

字段：
- `id`：主键
- `item_id`：外键到 `items.id`
- `filename`：文件名
- `url`：访问地址
- `created_at`：创建时间

关联：
- N:1 `items`

## 10. operation_logs（操作日志表）

字段：
- `id`：主键
- `user_id`：外键到 `users.id`
- `action`：操作类型
- `details`：操作详情
- `created_at`：创建时间

关联：
- N:1 `users`

## 11. 关键关系总览

- 用户与房屋权限：`users` N:M `houses`（`user_accessible_houses`）
- 用户默认房屋：`users.default_house_id` -> `houses.id`
- 房屋与房间：`houses` 1:N `locations`
- 分类与物品：`categories` 1:N `items`
- 房间与物品：`locations` 1:N `items`
- 标签与物品：`tags` N:M `items`（`item_tags`）
- 用户与业务数据：`users` 1:N `categories/tags/locations/items/operation_logs`
