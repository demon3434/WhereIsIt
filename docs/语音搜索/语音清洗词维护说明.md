# 语音清洗词维护说明

适用范围：
- 服务端语音搜索的文本清洗
- 词库文件：`prefix.txt`、`inline.txt`、`trailing.txt`

生效位置：
- 默认词库：`app/services/voice_search/cleaning_lexicon_defaults/`
- 用户持久化词库：`/opt/docker/whereisit/data/voice-cleaning-lexicon/`

## 1. 三类词库的用途

### `prefix.txt`
只在句首清洗。

适合放：
- 动作前缀：`帮我找`、`给我查一下`、`麻烦你帮我搜`
- 口头前缀：`那个`、`这个`、`就是`
- 明显的请求前导语：`请帮我`、`能不能帮我`

不适合放：
- 可能出现在物品名中的普通词
- 会和真实名词组成新词的单字

示例：
- `帮我找鼠标` -> `鼠标`
- `麻烦你帮我找一下罗技鼠标` -> `罗技鼠标`

### `inline.txt`
在整句任意位置清洗。

适合放：
- 催促语：`赶紧`、`马上`、`快点`
- 礼貌语：`麻烦你`、`请帮我`
- 明显无业务含义的插入语：`我想`、`我想要`

不适合放：
- 可能出现在真实物品名里的词
- 位置敏感词，比如 `下`

示例：
- `给我赶紧找鼠标` -> `鼠标`
- `我想看看那个小米手机` -> `小米手机`

### `trailing.txt`
只在句尾清洗。

适合放：
- 疑问尾巴：`在哪里`、`在哪儿`、`有吗`
- 语气尾巴：`吧`、`啊`、`呢`
- 方言尾巴：`咧`、`咯`

不适合放：
- 真实结尾词，尤其是物品名称尾词

示例：
- `扫地机器人在哪里` -> `扫地机器人`
- `鼠标有吗` -> `鼠标`

## 2. 维护原则

1. 优先加整词，不要轻易加单字。  
单字最容易误删真实物品名。

2. 优先加高频口语，不要把业务词当噪音词。  
例如 `鼠标`、`水池`、`瓶子` 这类名词不能进清洗词库。

3. 对方言词先保守处理。  
先加明显无业务含义的代词、语气词、请求词，不要一开始就加过多动词单字。

4. 有歧义的词尽量放 `prefix` 或 `trailing`，少放 `inline`。  
`inline` 影响范围最大，最容易误伤。

5. 每次新增词后，至少验证 2 类句子：  
- 一类是希望被清洗的句子  
- 一类是包含真实物品名的句子

## 3. 不建议乱加的词

以下类型默认不要直接加入：
- 单字动词：`找`、`查`、`看`、`拿`、`放`
- 方位单字：`上`、`下`、`里`、`外`
- 常见名词：`水池`、`瓶子`、`手机`、`鼠标`
- 容易构成真实物品名的词：`下水`、`上衣`、`机箱`

原因：
- 这些词可能本身就是物品名的一部分
- 会造成误删，降低召回准确率

## 4. 推荐流程

1. 先把候选词写进用户持久化词库，不直接改默认词库
2. 用几条真实口语测试清洗结果
3. 确认没有误删后，再决定是否回收进默认词库

## 5. 当前实现说明

- 服务端不是只在启动时一次性加载词库
- 当前是“懒加载 + 内存缓存 + 文件变更自动重载”
- 词库文件修改后，不需要重启服务即可生效
